Лабораторная работа №7: Многопоточность в Java

Описание

Данная лабораторная работа посвящена изучению многопоточности в Java.  
В ходе выполнения работы были реализованы программы для параллельных вычислений, поиска максимального элемента матрицы и моделирования работы грузчиков на складе.

Программа демонстрирует создание потоков, синхронизацию потоков, использование методов start() и join(), а также работу с механизмами ReentrantLock и Condition.

Ключевые концепции

Thread — создание и выполнение отдельных потоков

&nbsp;&nbsp;&nbsp;&nbsp;Многопоточность — параллельное выполнение задач

&nbsp;&nbsp;&nbsp;&nbsp;start() — запуск потока

&nbsp;&nbsp;&nbsp;&nbsp;join() — ожидание завершения потока

&nbsp;&nbsp;&nbsp;&nbsp;ReentrantLock — механизм блокировки потоков

&nbsp;&nbsp;&nbsp;&nbsp;Condition — ожидание и уведомление потоков

Задания:

1 Параллельное вычисление суммы массива

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Создан класс SumThread для вычисления суммы части массива.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Массив разделяется между двумя потоками.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Каждый поток вычисляет частичную сумму элементов.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;После завершения потоков результаты объединяются.

2 Поиск максимального элемента матрицы

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Создан класс RowMaxThread для поиска максимума в строке матрицы.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Для каждой строки создаётся отдельный поток.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;После завершения работы потоков определяется глобальный максимум матрицы.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Используется метод join() для ожидания завершения вычислений.

3 Моделирование работы склада

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Создан класс Warehouse для хранения товаров.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Реализован класс Loader, представляющий грузчика в отдельном потоке.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Используется ReentrantLock для синхронизации доступа к складу.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Используются Condition и методы await()/signalAll() для ожидания условий.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Грузчики переносят товары до достижения суммарного веса 150 кг.

4 Работа с потоками

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Продемонстрировано создание пользовательских классов потоков.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Реализовано параллельное выполнение вычислений.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Показана синхронизация потоков и безопасная работа с общими ресурсами.

5 Демонстрация работы программы

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Продемонстрирована параллельная обработка массива и матрицы.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Показана работа многопоточной системы склада.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Реализовано взаимодействие потоков через блокировки и условия ожидания.
